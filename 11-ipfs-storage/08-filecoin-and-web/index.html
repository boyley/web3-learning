🗄️ Filecoin 与去中心化 Web 生态

IPFS 解决了「内容怎么寻址、怎么分发」，但没解决「谁出钱保证它长期不丢」。Filecoin 在 IPFS 之上加了一层经济激励：客户付费，存储提供者用密码学证明「我确实一直存着你的数据」，存不好就没收抵押。这是本页的静态讲解，无需联网。

IPFS vs Filecoin：分工

维度IPFSFilecoin
本质P2P 内容寻址 + 分发协议去中心化存储【市场】+ 区块链
解决什么「内容是什么 / 怎么取」「谁来长期存、凭什么信他没删」
激励无（靠自愿 pin）有（付费 + 抵押 + 罚没）
持久性保证弱（没人 pin 就消失）强（存储交易 + 每日时空证明）
寻址CID同样用 CID（多数 Filecoin 节点也是 IPFS 节点）
一句话：IPFS 负责「找到内容」，Filecoin 负责「保证内容一直在」。两者互补，常一起用。

Filecoin 怎么保证「没删你的数据」

💰 存储交易（Storage Deal）

客户和存储提供者（SP）在链上达成交易：存多大、存多久、多少钱。SP 需锁定 FIL 抵押。

🔐 复制证明 PoRep

SP 首次存入时证明「我确实为你的数据生成了独立的物理副本」，防止假装存了。

⏱️ 时空证明 PoSt

SP 需持续、每天提交密码学证明，表明数据一直在。漏证/造假 → 抵押被罚没。

📥 检索（Retrieval）

取回数据可走 IPFS/网关，或走 Filecoin 检索市场；数据用 CID 寻址，和 IPFS 一致。

典型分层实践（把本合集串起来）

NFT / dApp 常见做法：
① 图片 + 元数据 → 上传并 pin 到 Pinata/Storacha（保证随时可取，模块 05/06）
② 重要资产再让服务把它落进 Filecoin 存储交易（长期归档、经济担保，本模块）
③ 链上合约 tokenURI 只存 ipfs://CID（省 gas、不可篡改，模块 06）
→ 热数据靠 IPFS pin 快速可取，冷/重要数据靠 Filecoin 长期兜底。
