② 真实最小接入代码（@walletconnect/ethereum-provider）

拿到 projectId（cloud.reown.com 免费申请）后，真实项目里这样接入。拿到的 provider 仍是 EIP-1193，前 9 个模块的 API 全通用。

import { EthereumProvider } from '@walletconnect/ethereum-provider'

// 1) 初始化：申请 projectId、指定链、开启内置二维码弹窗
const provider = await EthereumProvider.init({
  projectId: 'YOUR_PROJECT_ID',   // 来自 https://cloud.reown.com
  chains: [11155111],             // Sepolia 测试网（本系列只用测试网）
  showQrModal: true               // 自动弹出二维码 / 钱包列表
})

// 2) 连接：弹二维码，等待手机钱包扫码并批准
await provider.connect()

// 3) 之后就是熟悉的 EIP-1193，一切照旧
const accounts = await provider.request({ method: 'eth_accounts' })
const chainId  = await provider.request({ method: 'eth_chainId' })

// 签名同样通过 request（请求会推送到手机钱包确认）
const sig = await provider.request({
  method: 'personal_sign',
  params: ['0x...消息...', accounts[0]]
})

// 监听事件也和注入式一样
provider.on('accountsChanged', (a) => console.log('账户变化', a))
provider.on('chainChanged',    (c) => console.log('链变化', c))
provider.on('disconnect',      ()  => console.log('会话断开'))

进阶：正式项目常用 Reown AppKit（前 Web3Modal），createAppKit({ projectId, networks }) 一把梭，自带钱包列表与多链 UI，也支持 CDN 引入。

③ 签名安全自查清单

☠️ 最致命的坑：Permit / Permit2 链下授权。 它是 eth_signTypedData_v4 的一段签名， 不产生链上交易、钱包里看不到 pending，却等于把「花你代币的权限」签给了别人。攻击者事后随时能把你的币划走。 凡是「免 gas 领空投 / 一键授权」的签名，逐字段核对 spender 和 value，看不懂就拒绝。
🚫 eth_sign 盲签永远不要签。 它对任意 32 字节哈希签名，你无从得知内容（可能就是一笔转账的哈希），钱包会显示红色警告。
⚠️ 无限授权慎点。 ERC-20 的 approve(spender, uint256 max) 或 ERC-721 的 setApprovalForAll(op, true)， 一旦 spender 恶意，你的资产长期敞口。优先有限额度，定期上 revoke.cash 撤销。
